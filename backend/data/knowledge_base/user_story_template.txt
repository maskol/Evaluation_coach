EXAMPLE (SIMPLE, FICTIONAL)

USER STORY TITLE:
View Connection Quality Dashboard

USER STORY STATEMENT:
As a network administrator
I want to view a real-time dashboard of connection quality metrics
So that I can quickly identify and respond to connectivity issues

STORY DESCRIPTION:
What:
A web-based dashboard that displays key connection quality metrics including 
latency, packet loss, and bandwidth utilization for all active connections.

Why:
Network administrators need quick visibility into connection health to 
proactively address issues before they impact customers.

Who:
Network operations team, NOC staff, technical support personnel.

ACCEPTANCE CRITERIA:
AC1: Dashboard displays real-time metrics updated every 30 seconds
AC2: Users can filter connections by region, customer, or service type
AC3: Metrics include latency (ms), packet loss (%), and bandwidth (Mbps)
AC4: Dashboard loads within 3 seconds with up to 1000 active connections

TECHNICAL NOTES:
- Use WebSocket connection for real-time updates
- Cache aggregated metrics in Redis for performance
- API endpoint: GET /api/connections/metrics
- Frontend: React with Chart.js for visualizations

DEPENDENCIES:
- Metrics collection service must be deployed (Story #234)
- Access to production monitoring database

STORY POINTS:
5

DEFINITION OF DONE:
- [x] Code complete and peer reviewed
- [x] Unit tests written (>80% coverage)
- [x] Acceptance criteria validated by PO
- [x] Dashboard responsive on desktop and tablet
- [x] Documentation updated with screenshots
- [x] Deployed to staging environment
